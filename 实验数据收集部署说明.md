# 实验数据收集部署说明

## 阶段1：手动创建数据库集合

### 步骤
1. 登录微信小程序后台
2. 进入「云开发」→「数据库」
3. 创建以下5个新集合：

| 集合名称 | 用途 | 权限建议 |
|----------|------|---------|
| `user_sessions` | 用户会话记录 | 所有用户可读，仅创建者可写 |
| `answer_details` | 答题明细记录 | 所有用户可读，仅创建者可写 |
| `llm_interactions` | LLM交互日志 | 所有用户可读，仅创建者可写 |
| `terminology_answers` | 术语题答案 | 所有用户可读，仅创建者可写 |
| `session_statistics` | 会话统计汇总 | 所有用户可读，仅创建者可写 |

### 索引创建（可选，建议创建）

**user_sessions**:
- `session_id` (唯一索引)
- `group`
- `status`
- `created_at`

**answer_details**:
- `session_id`
- `group`
- `question_id`
- `created_at`

**llm_interactions**:
- `session_id`
- `group`
- `call_id`
- `event_type`
- `created_at`

**terminology_answers**:
- `session_id`
- `group`
- `question_id`
- `created_at`

**session_statistics**:
- `session_id` (唯一索引)
- `group`
- `created_at`

### 步骤
1. 在微信开发者工具中

- 控制台显示上传成功
- 云函数列表中出现 `submitLog`

## 阶段3：测试数据收集

### 测试实验组
1. 修改 `app.json`，将 `pages/questionnaire/questionnaire` 放在第一位
2. 点击「编译」
3. 填写问卷
4. 检查数据库：
   - `user_sessions`：应有一条记录，group='experimental'
   - `answer_details`：应有答题记录
   - `llm_interactions`：应有LLM调用记录

### 测试对照组
1. 修改 `app.json`，将 `pages/questionnaire/control` 放在第一位
2. 点击「编译」
3. 填写问卷
4. 检查数据库：
   - `user_sessions`：应有一条记录，group='control'
   - `answer_details`：应有答题记录
   - `llm_interactions`：应为空（对照组无LLM）

## 数据导出

### 方法1：通过小程序后台
1. 进入云开发 → 数据库
2. 选择集合
3. 点击「导出」

### 方法2：通过代码
```javascript
// 在云函数中执行
const db = cloud.database();
const data = await db.collection('user_sessions').get();

// 保存到云存储
await cloud.uploadFile({
  cloudPath: `exports/user_sessions_${Date.now()}.json`,
  fileContent: JSON.stringify(data)
});
```

## 常见问题

### Q1: 数据没有写入数据库？
**A**: 检查以下几点：
1. 云函数是否部署成功
2. 数据库集合是否创建
3. 权限设置是否正确
4. 查看云函数日志，看是否有报错

### Q2: 对照组为什么没有 LLM 记录？
**A**: 正常现象，对照组不调用 LLM 功能，所以 `llm_interactions` 为空。

### Q3: 如何清除测试数据？
**A**: 在数据库中直接删除记录，或创建一个测试用的 session_id 前缀（如 `test_`），方便过滤删除。

## 数据分析准备

### 核心指标计算公式

| 指标 | 计算公式 | 数据来源 |
|------|---------|---------|
| 完成率 | completed_sessions / total_sessions | user_sessions |
| 中途退出率 | abandoned_sessions / total_sessions | user_sessions |
| 漏填率 | unfulfilled_count / total_questions | user_sessions |
| 平均填写时长 | avg(duration) where status='completed' | user_sessions |
| 无效答案率 | invalid_answers / total_answers | answer_details |
| 逻辑矛盾率 | contradiction_count / total_answers | answer_details |
| LLM触发次数 | count(*) where group='experimental' | llm_interactions |
| LLM响应时长 | avg(response_time) where group='experimental' | llm_interactions |

### 分析工具建议

**Excel/Google Sheets**：
- 导出数据到CSV
- 使用数据透视表分析
- 绘制对比图表

**Python pandas**：
```python
import pandas as pd

# 读取数据
experimental = pd.read_json('experimental.json')
control = pd.read_json('control.json')

# 计算指标
exp_completion = experimental['status'].value_counts(normalize=True)['completed']
ctrl_completion = control['status'].value_counts(normalize=True)['completed']

print(f'实验组完成率: {exp_completion:.2%}')
print(f'对照组完成率: {ctrl_completion:.2%}')
```

- 进行t检验、卡方检验
