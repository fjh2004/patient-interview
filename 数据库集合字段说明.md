# 数据库集合字段说明文档（最终简化版）

本文档说明了实验所需的所有数据库集合及其字段。

---

## 集合列表（共3个）

### 1. user_sessions（用户会话集合）

**用途**：记录用户会话信息（应答率和时长指标）

| 字段名 | 类型 | 说明 | 必填 |
|--------|------|------|------|
| session_id | String | 会话ID（唯一标识） | 是 |
| user_id | String | 用户ID | 是 |
| group | String | 分组 "experimental" 或 "control" | 是 |
| status | String | 状态：in_progress/completed/abandoned | 是 |
| has_skip | Boolean | 是否有跳过（跳过必答题或空题） | 是 |
| start_time | Date | 开始时间（毫秒时间戳） | 是 |
| end_time | Date | 结束时间（毫秒时间戳，会话结束时更新） | 否 |
| duration | Number | 持续时长（毫秒） | 是 |
| exit_question_index | Number | 退出时的题目索引 | 否 |
| created_at | Date | 记录创建时间 | 是 |
| updated_at | Date | 记录更新时间 | 否 |

---

### 2. fill_record（问卷填写记录集合）

**用途**：存储用户提交的问卷答案（用于人工分析矛盾）

| 字段名 | 类型 | 说明 | 必填 |
|--------|------|------|------|
| record_id | String | 记录ID | 是 |
| answers | Object | 用户答案对象，键为题目ID，值为答案 | 是 |
| submit_time | Date | 提交时间 | 是 |
| status | String | 状态 "completed" | 是 |
| group | String | 分组标识 "experimental" 或 "control" | 是 |
| create_time | Date | 创建时间 | 是 |

---

### 3. llm_interactions（LLM交互记录集合）

**用途**：记录LLM交互（仅实验组，用于统计修正率）

| 字段名 | 类型 | 说明 | 必填 |
|--------|------|------|------|
| session_id | String | 会话ID | 是 |
| event_type | String | 事件类型：logic_check / ai_question | 是 |
| user_action | String | 用户操作：accept / reject / ignore | 否 |
| created_at | Date | 记录创建时间 | 是 |

---

## 集合关系说明

```
questionnaire (模板)
    ↓
    ├─→ user_sessions (会话 - 应答率和时长)
    │       ↓
    │       └─→ llm_interactions (LLM交互 - 仅实验组修正率)
    │
    └─→ fill_record (原始提交记录 - 人工分析矛盾)
```

## 论文指标统计方法

### 一、应答率（两组对比）

| 指标 | 计算公式 | 数据来源 |
|------|----------|----------|
| completion_rate | status='completed'的记录数 / 发放总数 | 人工统计发放数 + 数据库查询user_sessions |
| abandonment_rate | status='abandoned'的记录数 / 发放总数 | 人工统计发放数 + 数据库查询user_sessions |
| skip_rate | has_skip=true的记录数 / 发放总数 | 人工统计发放数 + 数据库查询user_sessions |

**查询示例**：
```javascript
// 实验组完成率
const experimentalCompleted = await db.collection('user_sessions')
  .where({ group: 'experimental', status: 'completed' })
  .count();

const completionRate = experimentalCompleted / totalDistributed; // totalDistributed 为发放总数
```

---

### 二、准确性（两组对比）

| 指标 | 计算公式 | 数据来源 |
|------|----------|----------|
| contradiction_rate | 有矛盾的问卷数 / 发放总数 | 人工分析 fill_record.answers + 人工统计发放数 |

**说明**：
- 矛盾判断由医护人员人工分析 fill_record.answers
- 例如：第6题选"糖尿病"但第7题选"以上皆无" → 矛盾
- 需要人工逐份问卷检查逻辑一致性

---

### 三、填写时长（两组对比）

| 指标 | 计算公式 | 数据来源 |
|------|----------|----------|
| avg_duration | 所有完成问卷的总时长 / 完成问卷数 | 数据库查询user_sessions的duration字段 |
| median_duration | 排序后取中间值 | 数据库查询user_sessions的duration字段 |

**查询示例**：
```javascript
// 实验组平均时长（毫秒→秒）
const experimentalSessions = await db.collection('user_sessions')
  .where({ group: 'experimental', status: 'completed' })
  .get();

const totalDuration = experimentalSessions.data.reduce((sum, s) => sum + s.duration, 0);
const avgDuration = (totalDuration / experimentalSessions.data.length) / 1000; // 转换为秒

// 中位数时长
const durations = experimentalSessions.data.map(s => s.duration).sort((a, b) => a - b);
const medianDuration = durations[Math.floor(durations.length / 2)] / 1000; // 转换为秒
```

---

### 四、LLM交互（仅实验组）

| 指标 | 计算公式 | 数据来源 |
|------|----------|----------|
| llm_call_count | llm_interactions 总记录数 | 数据库查询 |
| llm_logic_check_count | event_type='logic_check' 的记录数 | 数据库查询 |
| llm_ai_question_count | event_type='ai_question' 的记录数 | 数据库查询 |
| corrected_rate | user_action='accept' 的次数 / event_type='logic_check' 的次数 | 数据库查询 |

**查询示例**：
```javascript
// LLM调用总数
const llmCallCount = await db.collection('llm_interactions').count();

// 逻辑检查次数
const logicCheckCount = await db.collection('llm_interactions')
  .where({ event_type: 'logic_check' })
  .count();

// AI问答次数
const aiQuestionCount = await db.collection('llm_interactions')
  .where({ event_type: 'ai_question' })
  .count();

// 修正率
const acceptCount = await db.collection('llm_interactions')
  .where({ event_type: 'logic_check', user_action: 'accept' })
  .count();

const correctedRate = acceptCount / logicCheckCount;
```

## 使用说明

### 实验组日志记录
- `user_sessions`：记录会话信息
- `llm_interactions`：记录LLM交互（logic_check, ai_question）
- `fill_record`：记录完整答案

### 对照组日志记录
- `user_sessions`：记录会话信息
- `fill_record`：记录完整答案
- `llm_interactions`：不记录（对照组无LLM功能）

---

## 需要手动创建的集合

在微信小程序后台 → 云开发 → 数据库 中创建以下集合：

1. **user_sessions** - 会话记录
2. **fill_record** - 问卷答案（已存在，无需创建）
3. **llm_interactions** - LLM交互记录（仅实验组）
