# 用户体验反馈功能部署说明

## 一、功能概述

在问卷提交成功后，增加用户体验反馈环节：
1. 提交成功后显示结果页面
2. 1.5秒后弹出提示框，邀请用户参与反馈
3. 用户点击"参与反馈"后跳转到专门的反馈页面
4. 用户填写4个问题的反馈后提交

### 2. 问卷页面
- `pages/questionnaire/questionnaire.js`：修改跳转URL，添加`group=experimental`参数

## 四、部署步骤

### 步骤1：创建数据库集合
在云开发控制台创建 `user_feedback` 集合，字段如下：

| 字段名 | 类型 | 说明 |
|--------|------|------|
| feedback_id | String | 反馈记录ID |
| group | String | 用户分组（experimental/control） |
| user_id | String | 用户ID |
| session_id | String | 会话ID（可选） |
| questionnaire_id | String | 问卷记录ID（可选） |
| satisfaction_score | Number | 满意度评分（1-10） |
| recommendation_level | String | 推荐意愿（A-E） |
| ai_comparison | String | AI功能比较（A-E） |
| ai_acceptance | String | AI接受度（A-E，仅实验组） |
| submit_time | Number | 提交时间戳 |
| created_at | Date | 创建时间 |
| updated_at | Date | 更新时间 |

### 步骤2：部署云函数
在微信开发者工具中：
1. 右键点击 `cloudfunctions/submitUserFeedback` 文件夹
2. 选择"上传并部署：云端安装依赖"
3. 等待部署完成

### 步骤3：更新user_sessions集合（可选）
在 `user_sessions` 集合中添加字段：
- `has_feedback`: Boolean（是否已提交反馈）
- `feedback_id`: String（关联的反馈ID）

## 五、测试流程

### 测试1：实验组流程
```
1. 填写问卷（实验组）
2. 提交问卷 → 跳转到结果页

```

## 七、常见问题排查

### 1. 提示框不显示
- 检查结果页面的`onShow`函数是否正确执行
- 检查`group`和`recordId`参数是否传递正确
- 检查是否今天已经提示过（每日只提示一次）

### 2. 反馈提交失败
- 检查云函数是否部署成功
- 检查`user_feedback`集合是否存在
- 检查必填字段是否完整

### 3. 页面跳转错误
- 检查`app.json`中是否注册了`user-feedback`页面
- 检查页面路径是否正确

## 八、在论文中的应用

### 数据分析维度
1. **满意度分析**：实验组 vs 对照组满意度对比
2. **接受度分析**：用户对AI功能的接受程度
3. **推荐度分析**：产品推广潜力评估
4. **功能价值验证**：通过AI功能比较验证AI的价值

### 研究价值
- 补充客观指标外的**主观用户体验数据**
- 验证AI功能的**用户接受度和价值**
- 为系统优化提供**直接的用户反馈**

## 九、注意事项

1. **用户疲劳**：问卷已较长，反馈问题尽量简洁
2. **数据质量**：注意样本偏差（愿意反馈的用户可能更积极）
3. **隐私保护**：反馈数据关联用户信息，注意数据安全
4. **用户体验**：反馈环节应为可选，不影响主要功能
